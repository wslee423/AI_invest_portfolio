# CONSTITUTION.md — PortfolioAI 헌법

> 최상위 권위 문서. 에이전트와 사람 모두 따른다.
> 개정 실행은 사람만 가능. 에이전트는 제안만 가능.

---

## 1. 프로젝트 목적

로그인 없이 성향 진단 설문을 완료하면 AI가 맞춤 투자 포트폴리오를 생성하고,
결과를 PDF로 다운로드할 수 있는 일회성 웹 서비스.
데이터는 서버에 저장되지 않으며 탭을 닫으면 소멸한다.

---

## 2. 문서 위계

충돌 시 상위 문서 우선.

```
CONSTITUTION.md
  ↓ CLAUDE.md
  ↓ AGENTS.md
  ↓ QUALITY_SCORE.md
  ↓ docs/product-specs/*.md
  ↓ docs/ARCHITECTURE.md
```

---

## 3. 에이전트 자율 범위

`AGENTS.md`의 "자율 결정 가능" 항목만 승인 없이 결정한다.
명시되지 않은 것은 자율 범위가 아니다.

---

## 4. 충돌 해결

| 상황 | 처리 |
|------|------|
| 에이전트 간 충돌 | Reviewer가 QUALITY_SCORE 미달 시 거부권 행사. 그 외는 사람 결정 |
| 에이전트 ↔ 문서 충돌 | 즉시 멈추고 사람에게 보고 |
| 문서 간 충돌 | 위계 순서 적용. 해결 불가 시 사람에게 보고 |

---

## 5. 불변 원칙

어떤 에이전트도, 어떤 상황에서도 아래 원칙을 어길 수 없다.

### 원칙 1. 투자 결과 무보장 고지 필수
포트폴리오 결과 화면과 PDF에 반드시 아래 문구를 표시한다.
> "본 서비스는 투자 참고 정보를 제공하며 수익을 보장하지 않습니다."

**검증:** 결과 UI와 PDF 템플릿에 `DisclaimerBanner`가 존재하고 숨김 처리가 없을 것.

### 원칙 2. 서버 데이터 무저장
사용자의 설문 응답, 성향 점수, 포트폴리오 결과를 서버(DB, 로그, 파일)에 저장하지 않는다.
데이터는 sessionStorage와 메모리에만 존재하며 탭을 닫으면 소멸한다.

**검증:** API Route가 사용자 데이터를 외부 저장소에 write하는 코드가 없을 것.

### 원칙 3. AI 판단 근거 표시 의무
AI가 생성한 포트폴리오에는 판단 근거(성향 점수, 리스크 지표, 배분 이유)를 함께 표시한다.

**검증:** OpenAI 응답 스키마에 `reasoning` 필드가 존재하고 UI와 PDF에 렌더링될 것.

### 원칙 4. OpenAI 호출 시 PII 미포함
OpenAI API 호출 페이로드에 이름·연락처 등 식별 가능한 개인정보를 포함하지 않는다.
익명화된 성향 점수와 수치 데이터만 전달한다.

**검증:** OpenAI 호출 페이로드에 `name`, `email`, `phone` 필드가 없을 것.
